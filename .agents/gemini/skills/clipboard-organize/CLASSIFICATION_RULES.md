# 剪藏分类规则与算法

## 分类流程概述

```
输入 → 标题分析 → 摘要分析 → 领域匹配 → 相似度计算 → 置信度评估 → 输出建议
```

## 1. 标题分析 (Title Analysis)

### 关键词提取规则
- 提取名词和名词短语作为主题关键词
- 识别内容类型关键词（教程、研究、新闻等）
- 排除常见停用词（的、了、在、是等）

### 内容类型判断 (基于标题)
| 关键词 | 内容类型 | 置信度 |
|--------|----------|--------|
| 教程、指南、入门、步骤、如何 | Tutorial | 高 |
| 研究、分析、报告、调查、趋势 | Research | 高 |
| 概念、定义、原理、基础 | Knowledge | 中 |
| 最新、发布、更新、新闻、动态 | News | 高 |
| 工具、软件、应用、插件 | Tool | 中 |
| 总结、回顾、心得、经验 | Summary | 中 |

### 标题模式匹配
- `[主题]教程/指南` → Tutorial类型
- `[主题]研究/分析` → Research类型
- `[主题]概念/定义` → Knowledge类型
- `最新[主题]动态` → News类型
- `[工具名]使用指南` → Tool + Tutorial类型

## 2. 摘要分析 (Summary Analysis)

### 内容深度评估
- **浅层内容**: 介绍性、概述性、新闻性 → 适合`50_资源/文章/`
- **中层内容**: 教程、指南、工具使用 → 适合`30_研究/教程/`或`50_资源/教程/`
- **深层内容**: 研究、分析、原理探讨 → 适合`30_研究/[主题]/`
- **概念性内容**: 定义、解释、基础知识 → 适合`40_知识库/`

### 关键词密度分析
计算以下关键词类别的出现频率：
1. **教程关键词**: 步骤、操作、配置、安装、使用
2. **研究关键词**: 数据、图表、实验、结果、分析
3. **概念关键词**: 定义、含义、原理、本质、特征
4. **新闻关键词**: 最新、发布、宣布、推出、更新

### 长度与结构分析
- **短摘要** (<100字): 可能为新闻或简单介绍
- **中摘要** (100-300字): 可能为教程或概述
- **长摘要** (>300字): 可能为研究或深度分析

## 3. 领域匹配 (Domain Mapping)

### 领域关键词映射表
| 关键词类别 | 对应领域 | 建议路径前缀 |
|------------|----------|--------------|
| AI、机器学习、深度学习 | [[15_领域/工作职业/AI技术]] | `30_研究/AI/` |
| 前端、JavaScript、React | [[15_领域/工作职业/前端开发]] | `30_研究/前端/` |

| 生产力、效率、时间管理 | [[15_领域/个人成长/生产力]] | `30_研究/生产力/` |
| 健康、运动、饮食、睡眠 | [[15_领域/个人成长/健康生活]] | `30_研究/健康/` |

### 多领域处理规则
如果内容涉及多个领域：
1. **主领域**: 关键词出现频率最高的领域
2. **次领域**: 其他相关领域，在frontmatter中作为`related_domains`
3. **交叉领域**: 如"AI前端开发" → 同时归类到AI和前端领域

## 4. 相似度计算 (Similarity Calculation)

### 与现有内容比较
1. **标题相似度**: 使用编辑距离或关键词重叠率
2. **主题相似度**: 比较关键词集合的重叠
3. **领域相似度**: 检查是否属于同一领域

### 重复内容检测
- **标题相似度 > 80%**: 可能为重复内容，建议合并
- **摘要相似度 > 70%**: 可能为相同内容的不同版本
- **完全重复**: 建议用户确认是否保留

### 相关内容发现
- **主题相关**: 建议链接到相关笔记
- **系列内容**: 识别是否为系列文章的一部分
- **补充内容**: 发现可以补充现有笔记的内容

## 5. 置信度评估 (Confidence Assessment)

### 置信度等级
- **高置信度** (>80%): 分类明确，路径清晰，直接归档
- **中置信度** (50-80%): 分类基本明确，但可能需要微调
- **低置信度** (<50%): 分类不明确，建议人工确认

### 置信度计算因素
1. **标题明确性** (权重: 30%)
2. **摘要信息量** (权重: 25%)
3. **领域匹配度** (权重: 20%)
4. **现有内容相似度** (权重: 15%)
5. **内容类型一致性** (权重: 10%)

### 低置信度处理规则
- **建议路径**: `00_收件箱/待处理/`
- **提供备选**: 给出2-3个可能的分类选项
- **请求确认**: 要求用户明确分类

## 6. 文件夹映射规则 (Folder Mapping Rules)

### 主分类规则
| 内容类型 | 主要路径 | 备选路径 | 适用条件 |
|----------|----------|----------|----------|
| 深度研究 | `30_研究/[主题]/` | `30_研究/综合/` | 分析深入、有数据支持 |
| 教程指南 | `30_研究/教程/` | `50_资源/教程/` | 步骤明确、可操作性强 |
| 概念知识 | `40_知识库/[分类]/` | `40_知识库/基础/` | 定义性、原理性内容 |
| 参考文章 | `50_资源/文章/[主题]/` | `50_资源/综合/` | 综合性、概述性内容 |
| 新闻动态 | `50_资源/新闻/[领域]/` | `50_资源/动态/` | 时效性强、新闻性内容 |
| 工具资源 | `50_资源/工具/[类型]/` | `50_资源/软件/` | 工具介绍、软件评测 |

### 路径生成规则
1. **主题标准化**: 将主题转换为小写英文或拼音
2. **目录检查**: 检查目标目录是否存在，不存在则创建
3. **文件命名**: 使用`YYYY-MM-DD-标题.md`格式
4. **冲突处理**: 如果文件已存在，添加`-v2`、`-v3`等后缀

### 特殊情况处理
1. **多主题内容**: 选择最主要主题，其他主题作为标签
2. **模糊内容**: 放入`00_收件箱/待处理/`并添加`#待分类`标签
3. **系列内容**: 创建系列文件夹，如`30_研究/AI/系列名称/`
4. **更新内容**: 如果是对已有内容的更新，链接到原内容

## 7. 标签生成规则 (Tag Generation Rules)

### 标签类型
1. **主题标签**: #AI #前端 #生产力 (基于内容主题)
2. **类型标签**: #教程 #研究 #新闻 (基于内容类型)
3. **领域标签**: #技术 #生活 #工作 (基于所属领域)
4. **难度标签**: #入门 #进阶 #专家 (基于内容难度)
5. **来源标签**: #网络剪藏 #英文翻译 #原创 (基于内容来源)

### 标签数量限制
- 最多5个标签
- 优先级: 主题标签 > 类型标签 > 领域标签 > 其他标签
- 避免重复或相似标签

## 8. 输出质量检查

### 自动检查项
- [ ] 标题与内容匹配度
- [ ] 归类路径合理性
- [ ] 标签数量和质量
- [ ] 总结简洁性
- [ ] 核心观点提炼准确性

### 人工确认条件
- 置信度 < 50%
- 涉及多个主要领域
- 与现有内容高度相似
- 内容类型模糊不清

## 9. 算法优化建议

### 短期优化
1. 收集分类错误案例，调整关键词权重
2. 增加领域关键词库
3. 优化相似度计算算法

### 长期优化
1. 使用机器学习模型进行自动分类
2. 建立用户反馈机制
3. 实现个性化分类规则
