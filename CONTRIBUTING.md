# 贡献指南

感谢您对 Jarvis - Obsidian × Claude Code 双核 AI 管家系统的兴趣！我们欢迎各种形式的贡献。

## 🎯 贡献方式

### 1. 问题报告
- **Bug 报告**: 使用 GitHub Issues 报告遇到的问题
- **功能建议**: 提出新功能或改进建议
- **文档问题**: 报告文档错误或不清晰之处

### 2. 代码贡献
- **修复 Bug**: 提交 Pull Request 修复已知问题
- **新功能**: 实现新功能或增强现有功能
- **性能优化**: 改进系统性能或资源使用

### 3. 文档改进
- **完善文档**: 补充缺失的文档内容
- **翻译**: 帮助翻译文档到其他语言
- **示例**: 添加使用示例或教程

### 4. 社区支持
- **回答问题**: 在 Issues 或 Discussions 中帮助其他用户
- **分享经验**: 分享使用经验和最佳实践
- **推广项目**: 帮助宣传项目，让更多人受益

## 🔧 技能开发指南

### 技能结构
每个技能包含以下文件：
```
skill-name/
├── SKILL.md            # 技能定义和文档
├── TEMPLATE.md         # 输出模板（可选）
└── EXAMPLES.md         # 使用示例（可选）
```

### 创建新技能
1. 在 `.agents/skills/` 下创建新目录
2. 创建 `SKILL.md` 文件，定义技能名称、描述和工作流程
3. 遵循现有技能的格式和约定
4. 测试技能功能
5. 提交 Pull Request

### 技能规范
- **名称格式**: `技能名称_简短描述` (英文+中文)
- **描述**: 清晰说明技能功能和适用场景
- **工作流程**: 分步骤描述技能执行逻辑
- **输出格式**: 定义标准输出格式
- **错误处理**: 包含异常处理逻辑

## 💻 代码规范

### 通用规范
- 使用中文注释和文档
- 保持代码简洁和可读性
- 遵循现有项目的代码风格
- 添加必要的错误处理

### 提交信息规范
```
类型: 简短描述

详细描述（可选）

- 修复了什么问题
- 新增了什么功能
- 其他相关信息
```

**类型**包括:
- `feat`: 新功能
- `fix`: Bug修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具变更

## 🧪 测试

### 测试要求
- 新功能应包含测试用例
- 修复 Bug 时应添加回归测试
- 保持现有测试的通过状态

### 测试方法
1. **手动测试**: 在 Obsidian 和 Claude Code 中测试功能
2. **集成测试**: 测试技能间的协作
3. **文档测试**: 确保文档与代码一致

## 📝 Pull Request 流程

### 提交前
1. 确保代码符合项目规范
2. 更新相关文档
3. 添加或更新测试
4. 确保本地测试通过

### 提交时
1. 创建清晰的 Pull Request 标题和描述
2. 关联相关 Issue (如有)
3. 提供测试结果和截图（如适用）
4. 说明变更的影响和迁移步骤

### 审查过程
1. 维护者会审查代码和文档
2. 可能需要修改或补充内容
3. 通过审查后合并到主分支

## ❓ 常见问题

A: 可以从简单的文档改进或 bug 修复开始，查看带有 "good first issue" 标签的 Issue。

### Q: 有开发讨论群吗？

### Q: 贡献有奖励吗？
A: 主要是开源社区的认可和学习机会，暂无物质奖励。
